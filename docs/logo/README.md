# ICON4Py logo

Scalable vector (SVG) logo for ICON4Py. The mark is a true 3D icosahedron — the
icosahedral base grid is ICON4Py's defining geometry — projected to 2D through a
parametrizable perspective camera. Markers identify the topological entities of the
mesh:

- **circles** at the 20 **cell centres** (triangular face centroids),
- **squares** at the 30 **edge midpoints**,
- nothing on the 12 vertices.

Each marker is a flat shape lying in the tangent plane at its position in 3D and only
then projected, so it is foreshortened with the correct perspective on faces tilted away
from the viewer. Faces are solid-shaded and hidden (back-facing) faces and markers are
culled.

| File | Contents |
| --- | --- |
| `icon4py_icon.svg` | Icosahedron mark only (use as app icon / favicon). |
| `icon4py_logo.svg` | Mark + `ICON4Py` wordmark. |

## Regenerating

The logo is generated by [`scripts/python/generate_logo.py`](../../scripts/python/generate_logo.py)
and is fully scriptable — every feature is a flag and the camera is configurable. Run it
through the unified dev-scripts entry point:

```bash
# Default logo (writes both files into this directory):
./scripts/run generate-logo

# Custom viewpoint and stronger perspective:
./scripts/run generate-logo --azimuth 70 --elevation 35 --distance 3.0

# See all options:
./scripts/run generate-logo --help
```

### Useful flags

| Flag | Effect |
| --- | --- |
| `--azimuth / --elevation / --roll` | Camera orientation (degrees). |
| `--distance` | Camera distance; smaller means stronger perspective. |
| `--palette {atmosphere,vibrant,mono}` | Colour scheme (default `atmosphere`). |
| `--faces / --wireframe` | Solid shaded faces or wireframe. |
| `--cell-centers / --no-cell-centers` | Toggle the cell-centre circles. |
| `--edge-midpoints / --no-edge-midpoints` | Toggle the edge-midpoint squares. |
| `--vertices / --no-vertices` | Toggle vertex dots (off by default). |
| `--wordmark / --no-wordmark` | Whether to also write the wordmark variant. |
| `--background` | Background colour, or `none` for transparent (default). |

The script depends only on NumPy and `drawsvg` (declared in the `scripts` dependency
group), so no model packages need to be installed to regenerate the logo.
