# icon4py-atmosphere-amps

GT4Py port of the AMPS spectral bin (habit-predicting) microphysics scheme
(Hashino & Tripoli), ported from the Fortran implementation embedded in
SCALE (contrib/AMPS). Work in progress; see the design spec in the
scale_amps repository: docs/superpowers/specs/2026-07-16-amps-icon4py-port-design.md.

## Module map (M1)

Paths below are relative to this package root
(`model/atmosphere/subgrid_scale_physics/amps/`).

| Module                                                         | Purpose                                                                                                                                                                                                                                                                                                                                                                                        |
| -------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `src/.../amps/core/constants.py`                               | Physical constants for AMPS microphysics in CGS units (`AmpsConst`), transcribed from `mod_amps_const.F90`.                                                                                                                                                                                                                                                                                    |
| `src/.../amps/core/thermo.py`                                  | Saturation-vapor-pressure tables/accessors, thermodynamic coefficient functions, and the theta_il closure; numpy array-in/array-out plus one DSL table-accessor deliverable (`esat_lk_dsl`).                                                                                                                                                                                                   |
| `src/.../amps/core/bin_grid.py`                                | Liquid (rain+haze) and ice bin-boundary construction (`binmicrosetup_scale`, the `binb(i+1) = a_b*binb(i) + b_b` recurrence).                                                                                                                                                                                                                                                                  |
| `src/.../amps/config.py`                                       | `AmpsConfig`: one frozen dataclass mirroring both Fortran namelists (`/AMPS_param/` and `PARAM_ATMOS_PHY_MP_AMPS_bin`), including the `cloudlab()` constructor used by the reference-data driver.                                                                                                                                                                                              |
| `src/.../amps/core/index_maps.py`                              | PPV (per-particle-variable) index maps and true compile-time dimension parameters, transcribed verbatim from `par_amps.F90` and `maxdims.F90`.                                                                                                                                                                                                                                                 |
| `src/.../amps/core/lookup_tables.py`                           | Loaders for the collision-efficiency LUTs, habit-frequency tables, diagnostic maps, `lmt_mass_*`, and `znorm`, plus computed-table generators; reads the packaged npz archives via `importlib.resources`.                                                                                                                                                                                      |
| `codegen/convert_luts.py` + `src/.../amps/codegen/generate.py` | `convert_luts.py` is the offline converter from raw AMPS_DATA Fortran table files to the two packaged npz archives under `src/.../amps/data/`; `codegen/generate.py` is the runtime code-GENERATION driver (`emit_module`/`check_regenerated`) that writes committed DSL source under `core/generated/`, enforcing the split-operator (\<=8 bins per `@gtx.field_operator`) design constraint. |
| `src/.../amps/state.py`                                        | Per-column state bundles (`LiquidState`, `IceState`, `AerosolState`, `ThermoState`) mirroring the Fortran qrpv/qipv/qapv property-vector-bin arrays, plus numpy \<-> gt4py `(Cell, K)` field conversion.                                                                                                                                                                                       |
| `src/.../amps/core/packing.py`                                 | SI (SCALE) \<-> CGS-mixing-ratio (AMPS) state packing/unpacking (`pack_scale_to_amps`, `unpack_amps_to_scale`), transcribed from `Z_LOOP_01` and the tendency/energy blocks.                                                                                                                                                                                                                   |
| `src/.../amps/core/rng.py`                                     | Counter-based RNG for AMPS habit selection, bit-exact between its numpy replica and DSL field-operator form, graduated from the M0 spike.                                                                                                                                                                                                                                                      |
| `src/.../amps/utils.py`                                        | Small standalone utilities (`recursion_limit`), amps' own copy of the pattern already used by `muphys`, kept dependency-free rather than imported.                                                                                                                                                                                                                                             |
| `src/.../amps/driver/ref_data.py`                              | Typed reference-data reader for the M0 instrumented-Fortran binary dumps (`MicroRecord`/`SedRecord`), a typed port of `scale_amps/scripts/amps_dump_reader.py`.                                                                                                                                                                                                                                |
| `src/.../amps/driver/box.py`                                   | Box-driver skeleton: `BoxCase` bundles a runnable single-column case; `case_from_micro_record` replays a dumped reference record into one; `run_box` is the (unimplemented) M2 per-timestep entry point.                                                                                                                                                                                       |

Status: M1 foundations complete; M2 warm-phase next.

See also, in the scale_amps repository: the design spec
(`docs/superpowers/specs/2026-07-16-amps-icon4py-port-design.md`) and the
Fortran-source mapping (`docs/superpowers/facts/m1/fortran-mapping.md`).
